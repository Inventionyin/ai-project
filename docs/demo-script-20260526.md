# AI Test Platform 演示路线

目标时长：5-10 分钟

适用场景：给负责人、需求方或测试负责人展示当前验收版本。

## 演示前准备

1. 确认平台可访问。
2. 准备一个可登录账号。
3. 确认项目中已有导入数据、接口集合、测试套件和运行记录。
4. 打开以下证据链接备用：
   - CI：`https://github.com/Inventionyin/ai-project/actions/runs/26447572998`
   - real-e2e：`https://github.com/Inventionyin/ai-project/actions/runs/26447572996`
   - 复跑证据：`docs/operations-rerun-20260526.md`

## 推荐演示流程

### 1. 登录和首页

说明平台是内部测试工作台，默认以邮箱和密码登录。登录后进入项目工作区。

讲法：

> 这里不是普通后台模板，而是围绕测试资产、AI 治理、自动化执行和验收报告收口的工作台。

### 2. 仪表盘

进入 `仪表盘`。

展示点：

- 今日执行、通过、失败、进行中。
- 时间范围和统计维度下拉筛选。
- 自定义仪表盘模块开关。
- 质量门禁、失败 Top 5、最近运行。

讲法：

> 仪表盘用于项目负责人快速看质量趋势，也支持按套件或用例切换维度。

### 3. 资产中心

进入 `资产中心`。

展示点：

- 需求管理。
- 用例管理。
- 接口管理。
- 测试套件。
- 统一资产操作入口。

讲法：

> 资产中心只保留需求、用例、接口、套件这些核心资产，AI 和配置类能力已从主资产区拆开，界面不会继续堆功能。

### 4. AI 治理

进入 `AI能力 -> 自动生成测试用例` 或 `AI能力 -> 用例治理`。

展示点：

- 文档检查。
- 候选用例生成。
- 人工确认。
- 正式入库。
- 重复、低价值、P0 覆盖不足等治理建议。

讲法：

> AI 不直接改正式数据，先生成建议，再由人确认应用，避免自动化乱改资产。

### 5. 接口调试

进入 `资产中心 -> 接口管理`，打开一个接口集合详情。

展示点：

- API 调试台。
- 选择环境。
- 运行请求。
- 查看状态码、耗时、响应头、响应体。
- 保存为用例绑定。
- 加入测试套件。

讲法：

> 这里已经不是只能看接口资产，而是可以从接口集合一路走到回归套件。

### 6. 测试套件执行

进入 `资产中心 -> 测试套件`。

展示点：

- 套件列表。
- 默认环境。
- 点击运行。
- 跳转到 Run 详情页。

讲法：

> 测试套件是回归用例集合，运行后会生成可追踪的 Run 记录。

### 7. 验收中心

进入 `设置 -> 验收中心`。

展示点：

- 总体状态。
- 检查项。
- 外部系统。
- 核心指标。
- 下一步。
- 报告预览。
- 复制汇报口径。
- 下载报告。
- 验收演示路线提示。

讲法：

> 验收中心负责把功能、数据、联调和运维证据收成一份能汇报的报告。

### 8. CI/E2E 和运营证据

打开 GitHub Actions 链接。

展示点：

- CI 成功。
- real-e2e 成功。
- 外部联调、生产检查、性能基线、备份恢复证据。

讲法：

> 代码不是只在本地跑通，已经放进 CI 和真实联调证据链里。

## 收尾话术

> 当前平台已经达到本阶段验收版本。后续主要不是继续堆新功能，而是按真实项目节奏持续导入数据、定期跑性能基线、定期做备份恢复演练，并在客户环境变化后复跑外部系统闭环。
